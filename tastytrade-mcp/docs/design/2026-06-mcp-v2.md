# Tastytrade MCP v2 — Technical Design

**Status:** In progress · **Branch:** `mcp-server-refactor` · **Date:** 2026-06

## 1. Background & motivation

The v1 server is a thin wrapper: 23 tools in a single `src/server.py`, each mapping ~1:1 onto a
Tastytrade REST endpoint and returning the raw JSON (pretty-printed) back to the model. It works, but
it inherits every problem the Honeycomb team documented in
[*"MCP, Easy as 1-2-3?"*](https://www.honeycomb.io/blog/mcp-easy-as-1-2-3):

- **Token volume is the #1 problem.** API JSON is shaped for UIs and code, not for an LLM's context
  window. A full option chain or positions payload floods context with fields the model never uses.
- **LLMs doom-loop on unfamiliar request shapes.** Without auto-correction and guided errors, a model
  that gets an order leg or chain filter slightly wrong fails, tweaks, fails again, then "drastically
  simplifies the query into something useless."
- **No curation.** Every field, every null, every internal id is passed through. No summaries, no
  derived stats, no caching.

Honeycomb's conclusion — *build curated tools for LLMs, not API wrappers* — is the thesis of this
refactor. We port their **techniques** (few high-level primitives, schema-rich params with
auto-correction, suggestion-bearing errors, server-side summaries, TTL caches with locally-served
pagination, an eval harness) onto the Tastytrade domain.

## 2. Goals / non-goals

**Goals**
- Consolidate 23 endpoint-wrappers into ~12 task-oriented tools.
- Cut response tokens via minimal shapes + computed `summary` blocks.
- Make tools resilient to LLM mistakes (auto-correction) and instructive on failure (guided errors).
- Add a TTL cache; serve pagination/search/sort from cache.
- Make order placement explicitly safe (env gate + confirm).
- Ship an eval harness and a Harbor benchmark comparing v1 (main) vs v2 (this branch).

**Non-goals**
- Streaming market data (DXLink) — dropped `get_quote_token`.
- Changing the OAuth/transport model — still FastMCP + stdio + httpx + OAuth refresh.
- Real-money benchmarking — benchmark runs against recorded mock fixtures.

## 3. Current state (v1)

23 tools across accounts/portfolio, orders, market-data/instruments, watchlists. Raw passthrough via
`_fmt` / `_items`. No resources/prompts, no logging, no rate-limit handling, no validation, no cache
beyond the account list, global client singleton. 45 unit tests, 80% coverage gate.

## 5. Architecture

Keep FastMCP + Python 3.13 + httpx + stdio; add `pydantic>=2`.

```
src/
  server.py            # FastMCP init + instructions + register_all()
  client.py            # OAuth/transport + logging + 429 retry
  config.py            # Settings: TT_ENABLE_TRADING, LOG_LEVEL, cache TTLs
  infra/
    logging.py         # stderr structured logging (stdio-safe)
    errors.py          # @guarded_tool -> {"error","suggestions":[...]}; HTTP/Validation -> guidance
    correction.py      # pre-validation auto-correct: kebab/camel->snake, case-fold enums, str->int
    cache.py           # TTLCache (per-resource TTL, LRU) + CachedCollection (local paginate/search/sort)
    pagination.py      # PaginationParams + apply_pagination
  schemas/             # Pydantic = Zod analogue (model_validator before=correct, after=semantic)
    common.py orders.py chain.py
  shaping/             # minimal shapes + summary builders
    summarize.py portfolio.py chain.py transactions.py orders.py market_data.py
  tools/               # one module per domain, each register(mcp)
    accounts.py market_data.py options.py transactions.py orders.py watchlists.py
```

- **Pydantic schemas** are the Zod analogue. `model_validator(mode="before")` runs auto-correction;
  `mode="after"` runs semantic validation (the `superRefine` analogue). FastMCP accepts Pydantic
  models as tool parameters.
- **Error guidance** (`infra/errors.py`): a `@guarded_tool` decorator wraps every tool, catching
  `httpx.HTTPStatusError` and `ValidationError` and returning a JSON object with an `error` message and
  a `suggestions` array containing CORRECT/INCORRECT examples — never a raw traceback.
- **TTL cache** (`infra/cache.py`): per-resource TTLs (quotes 15s, metrics 120s, transactions/chain
  300s, accounts/instruments 900s, live orders no-cache), LRU cap. `CachedCollection` stores a fetched
  list so repeated paging/searching never re-hits the API.
- **Logging** (`infra/logging.py`): structured logs to **stderr** (stdout is the MCP channel) — tool
  name, duration, cache hit, response bytes.

## 6. Response shaping

Principle: every tool returns `{summary, ...items}`; decimal strings → rounded floats; nulls and
internal fields dropped; kebab-case → snake_case.

- `get_option_chain.summary`: `{atm_strike, n_strikes, dte, strike_range, iv_range, top_volume[],
  top_open_interest[]}`. The nested chain endpoint has no quotes — enrichment is a second **batched**
  `/market-data/by-type` call over the *filtered* symbols only (the curated value-add + token win).
- `get_portfolio.summary`: total P/L, day P/L, exposure by underlying. (Greeks rollup deferred — TT
  positions have no greeks; would need per-option enrichment.)
- `query_transactions.summary`: count, net cash effect, total fees, by-type counts, date range.
- `get_portfolio_history`: downsample to ≤30 points + start/end/change/max-drawdown; raw series dropped.

## 7. Order safety model

Two independent gates on `place_order` (decided with the user):
1. **`TT_ENABLE_TRADING`** env flag — default **off** → `place_order` refuses with guidance, preview &
   cancel still work.
2. **`confirm=true`** required parameter — even with trading enabled, a call without `confirm=true` is
   rejected with the previewed effect echoed back.

## 8. Eval strategy & Harbor benchmark

- **Mock fixture API** (`tests/fixtures/mock_api/`): a small ASGI app replaying scrubbed recorded TT
  responses, a fake `/oauth/token`, and an order endpoint that records submitted bodies for verifier
  assertions. Pointed at via `API_BASE_URL`. Shared by integration tests, evals, and Harbor.
- **Tier 1 (CI):** deterministic pytest evals feeding recorded LLM-misuse payloads through
  correction+validation; assert fixes and suggestion-bearing errors.
- **Tier 2 (`make eval`, manual):** YAML cases driving Claude via the Anthropic API against the MCP +
  mock API; track tool calls/tokens/latency, optional judge.
- **Harbor benchmark** (`benchmark/`): ~14 graded tasks, one `job.yaml` with two `agents:` entries —
  claude-code+v1 (main, via git tag) vs claude-code+v2 (this branch). Harbor natively records reward,
  phase timings, and token/cost; compare via `harbor view jobs`.

**v1 snapshot:** add a `[project.scripts]` entry (`tastytrade-mcp = "src.server:main"`) and tag
`v1-baseline` so both servers are launchable in one job.

## 9. Migration

Clean break on tool names (no aliases). MCP clients re-discover tools each session. v1 remains runnable
from the `v1-baseline` tag and from `main`.

## 10. Phased rollout

- **Phase 0** — foundations: config, infra (errors/cache/correction/pagination/logging), schemas,
  entry point, pydantic dep, design doc.
- **Phase 1** — consolidation: shaping helpers + the 12 tools + server split into `tools/` modules +
  rewritten tests; remove v1 tools.
- **Phase 2** — evals: mock fixture API + integration tests + tier-1 CI evals + tier-2 runner.
- **Phase 3** — Harbor benchmark: env + tasks + `job.yaml` + results.

Each phase is a PR into `mcp-server-refactor`. All four phases are merged.

## 10a. Results — local smoke comparison (v1 main vs v2)

A quick local comparison (both servers pointed at the mock API, same outcomes) shows the
core win is **fewer tool calls / agent round-trips** and **analysis-ready responses**:

| Task (same end result) | v1 calls | v2 calls |
|---|---|---|
| Total unrealized P/L | 2 (`get_balances` + `get_positions`, agent computes P/L) | 1 (`get_portfolio`, P/L precomputed in `summary`) |
| SPY 2026-04-17 chain w/ ATM + per-strike IV & volume | 3 (`get_option_expirations` + `get_option_chain` + `get_quote`, agent computes ATM) | 1 (`get_option_chain`, enriched + `summary`) |
| Market snapshot (quote + IV metrics) | 2 (`get_quote` + `get_market_metrics`) | 1 (`get_market_data`) |

Per-response **bytes** were comparable on these minimal fixtures (v2 slightly larger
because it adds computed summaries and inline quote-enrichment). The byte advantage of v2's
field-trimming only appears on realistic Tastytrade payloads, which carry dozens of fields
per object that v1 passes through verbatim. The authoritative, end-to-end comparison —
agent-loop tokens, tool-call counts, latency, and success rate across both servers — is the
Harbor benchmark in `benchmark/` (`harbor run -c benchmark/job.yaml`).

## 11. Decision log / deferred

- Greeks rollup in `get_portfolio` — deferred (needs per-option enrichment).
- UI deep-link tool (Honeycomb `get_trace_link` analogue) — skipped; TT web URL formats undocumented.
- Tier-2 evals — manual/nightly, not blocking CI (needs `ANTHROPIC_API_KEY`).
